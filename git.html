Git and Version Control System

1. Version Control System (VCS)

Version Control System is a tool that helps developers track and manage changes to source code over time.

  • Maintains multiple versions of a project.
  • Allows collaboration among multiple developers.
  • Supports rollback to previous versions if needed.

Centralized and Distributed Version Control Concept

2. What is Git?

Git is a distributed version control system developed by Linus Torvalds in 2005. It allows multiple developers to work on the same codebase simultaneously without overwriting each other’s work.

Git Workflow Overview

Features of Git

  • Distributed: Every developer has a complete local copy of the repository.
  • Speed: Git is optimized for performance.
  • Branching and Merging: Flexible branching strategy for parallel development.
  • Staging Area: Allows you to prepare commits before saving.
  • Collaboration: Easily integrates with GitHub, GitLab, and Bitbucket.
  • Open Source: Free and widely used across the world.

3. Git Workflow

The Git workflow defines the process of managing files through different states:

  • Working Directory: Contains project files being worked on.
  • Staging Area (Index): Holds changes before committing.
  • Local Repository: Stores committed code versions on the user’s system.
  • Remote Repository: Stores code on a remote server (e.g., GitHub).

Git Workflow: Working Directory → Staging → Local → Remote

4. Common Git Commands

Here are some frequently used Git commands:

git init # Create a new local repository git status # View status of files (modified/staged/untracked) git add <file> # Add file to staging area git commit -m "message" # Save changes to local repo git diff # Show changes between commits git clone <url> # Copy remote repo to local git push # Upload commits to remote repo git pull # Download latest changes from remote git branch # List all branches git config # Configure username and email

5. Branching and Merging

Branching allows developers to create isolated environments for new features, while merging integrates those changes into the main branch.

  • Create a branch to develop a feature separately.
  • After testing, merge the feature branch back to the main branch.
git branch new-feature # Create a new branch git checkout new-feature # Switch to the new branch git merge new-feature # Merge branch into main git branch -d new-feature # Delete the branch

6. Advantages of Git

  • Facilitates collaboration and teamwork.
  • Maintains complete version history of changes.
  • Supports distributed development.
  • Fast and efficient for large projects.
  • Reliable and open-source.

7. Git Hosting Services

  • GitHub: Cloud-based Git repository hosting with collaboration tools.
  • GitLab: Open-source Git platform for CI/CD and DevOps.
  • Bitbucket: Repository hosting service by Atlassian.
