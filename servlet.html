Introduction to Servlets and Web Servers

A **Servlet** is a Java class or web program that handles client requests, processes them (often interacting with databases), and replies back with a response, typically an HTML page. This is part of the Java EE stack.

Servlet Architecture Overview

When a web server receives a request, it hands it to the **Servlet Container** (like Tomcat), which manages the lifecycle and execution of the Servlets.

Servlet Architecture: Request flow via the web server and container.

CGI vs. Servlet

Servlets are a highly efficient replacement for traditional **Common Gateway Interface (CGI)** programs:

CGI vs. Servlet Comparison

  • **Basic:** CGI programs are written in native OS; Servlets use **Java**.
  • **Platform Dependency:** CGI is platform-dependent; Servlets are **portable** (platform-independent).
  • **Process Creation:** CGI creates **its own process** for each client request (slow); Servlets use **threads** within the JVM (faster).
  • **Execution:** CGI runs on a separate process; Servlets run on the **JVM**.
  • **Security/Speed:** Servlets are **Faster** and **More Resistant to Attacks** than CGI.

Servlet Life Cycle and Methods

The **Servlet Life Cycle** is managed by the Servlet Container and defines how a Servlet is loaded, initialized, served, and destroyed.

Servlet Life Cycle Steps

The entire lifecycle consists of the following phases:

  1. Servlet class is loaded.
  2. Servlet instance is created.
  3. **init() method is invoked.** (Called only once for initialization.)
  4. **service() method is invoked.** (Invoked at each client request.)
  5. **destroy() method is invoked.** (Invoked only once when the servlet is being destroyed.)

Visual representation of the Init-Service-Destroy sequence.

Methods of the Servlet Interface

The core javax.servlet.Servlet interface defines these methods that all servlets must implement:

public void init(ServletConfig config) // Initializes the servlet. Invoked only once.

public void service(ServletRequest request, ServletResponse response) // Provides response for the incoming request. Invoked at each request.

public void destroy() // Invoked only once, indicating the servlet is being destroyed.

public ServletConfig getServletConfig() // Returns the ServletConfig object.

public String getServletInfo() // Returns information about the servlet (writer, version, etc.).

Steps to Create a Servlet Example

The servlet example can be created in three ways:

  • By **implementing the Servlet interface**.
  • By **inheriting the GenericServlet class**.
  • By **inheriting the HttpServlet class** (most common for web applications).

Handling HTTP Request and Response

Servlet Execution Process and Container Role

The **web container** is responsible for handling the request through these steps:

  1. The container **loads the servlet class** and **instantiates the servlet class**.
  2. It **maps the request** with the servlet defined in the web.xml file.
  3. It creates **Request and Response objects** (HttpServletRequest and HttpServletResponse).
  4. It calls the **init() method** passing the ServletConfig object.
  5. It calls the **service() method** on a thread.
  6. The public service() method internally calls the protected service() method.
  7. The protected service() method calls specific methods (like **doGet()** or **doPost()**) depending on the HTTP request type.
  8. The doGet/doPost method generates the response, which is passed to the client.
  9. After sending the response, the web container **deletes the request and response objects**.

Reading and Initialization of Servlet Parameters

Servlet parameters can be read via the **ServletRequest** object (e.g., input from HTML forms) or initialized via the **ServletConfig** object (for initialization parameters).

Redirection

You can redirect the client to a different URL using the sendRedirect() method of the HttpServletResponse interface.

// Example of Redirection
response.sendRedirect("http://newurl.com/success");

Servlet Execution Process (Developer Steps)

  1. **Create servlet** (Source code).
  2. **Compile the servlet**: Javac -cp .:servlet-api.jar Demoservlet.java (Requires servlet-api.jar).
  3. **Create the deployment descriptor** (web.xml file).
  4. **Deployment**: Place the compiled .class file in the WEB-INF/classes folder.
  5. **Testing**: Place the application folder into the webapps directory of your server (e.g., C:\xampp\tomcat\webapps) and start the Tomcat service.

Session and Cookie Management

Session Tracking in Servlets

Session Tracking (or **session management**) is the way to **maintain state (data) of a user** across multiple client requests.

Session Tracking Techniques:

  1. **Cookies**
  2. **Hidden Form Field**
  3. **URL Rewriting**
  4. **HttpSession** (Most robust, relying on a unique session ID managed by the container).

Cookies

A **Cookie** is a small piece of textual information stored in key-value pair format in the client's browser. It is persisted between multiple client requests.

Types of Cookies:

  • **Non-persistent cookie:** Valid for a **single session only**. Removed when the user closes the browser.
  • **Persistent cookie:** Valid for **multiple sessions**. Removed only if the user logs out or signs out.

Cookie Code Examples:

// 1. Creating and Setting Max Age (Persistence)
Cookie ck = new Cookie("Name", name);
ck.setMaxAge(24*60*60); // Max age in seconds (1 day)
res.addCookie(ck); // Adding cookie to the response

// 2. Getting cookies
Cookie ck[] = request.getCookies();
for(int i=0; i<ck.length; i++) {
  out.print(ck[i].getName() + " " + ck[i].getValue());
}

// 3. Deleting a cookie
Cookie ck_delete = new Cookie("user", "");
ck_delete.setMaxAge(0); // Set max age to 0 seconds
response.addCookie(ck_delete);

HttpSession Interface

The container creates a **unique session ID** for each user to identify them. The HttpSession object is used to perform two tasks: **bind objects** and **view/manipulate session information** (ID, creation time, last accessed time).

Key HttpServletRequest Methods for Session:

public HttpSession getSession() // Returns current session or creates one if none exists.

public HttpSession getSession(boolean create) // Returns current session or creates one ONLY if 'create' is true.

Common HttpSession Methods:

  • public String getId(): Returns the unique identifier value.
  • public long getCreationTime(): Returns the session creation time.
  • public long getLastAccessedTime(): Returns the last time the client sent a request associated with this session.
  • public void invalidate(): Invalidates the session then unbinds any objects bound to it.
